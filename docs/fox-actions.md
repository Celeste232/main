# 🦊 여우 스킨 — 액션 매핑 & 제작 스펙

> 상태: **draft (매핑 확정 대기)**. 스크린샷은 코드로 못 자르니, 여기서 "어떤 동작을
> 어떤 액션 키에 넣을지"만 확정한다. 최종 시트 파일을 주면 그 다음은 기계적이다.

## 결정

- 여우 = Meow Mode의 **새 PNG 스킨** (`settings.catSkin: 'png'`). 기존 행동 엔진과
  슬라이스 파이프라인을 그대로 재사용한다 — 새 캐릭터 시스템을 만들지 않는다.
- 스프라이트는 `src/assets/cat/<액션키>/<n>.png` 로 들어가고 렌더러가 자동으로 집어간다.
  (`src/components/Cat/catFrames.ts` 의 `getFrames` + `import.meta.glob`)

## ⚠️ 제작 전 알아둘 제약 2개

1. **프레임 수** — 시트는 액션당 **3프레임**인데, 앱은 액션별로 다른 수(`FRAME_SPECS`,
   2~4)를 기대한다. → PNG 스킨은 "있는 프레임 수"만큼만 돌도록 `Cat.tsx`를 살짝 고친다
   (내가 함). 그러면 3프레임 그대로 OK.
2. **빠진 액션 폴백** — 여우 PNG가 없는 키는 현재 **고양이 SVG로 폴백**한다. 그대로 두면
   "여우인데 가끔 고양이로 변함" 버그. → 폴백을 **여우 `idle` 프레임**으로 바꾼다 (내가 함).
   따라서 `idle`은 반드시 여우 art가 있어야 한다.

### ➕ 여우엔 있는데 앱 키가 없음 → 지금은 드롭 (원하면 나중에 행동 추가)

굴 파기, 나뭇잎/나뭇가지 물고 다니기, 누워서 공놀기(→play-cursor로 대체).

### 🪵 배경 소품이 필요해 데스크탑에 안 맞음 → 드롭

창밖 구경(`S3·8`), 담요 덮고 숨기(`S3·9`), 나무 뒤 숨기(`S5·9`),
높은 곳 바라보기/그루터기(`S4·7`). (집·밥그릇은 prop이 있으니 OK, 나머지 소품은 없음)

### ⚠️ 앱 키인데 여우 art가 아직 없음 → `idle`로 폴백되거나, 나중에 그려야

`held`(들렸을 때), `knead`(꾹꾹이), `sneeze`, `pounce`, `sparkle`, `caught`,
`sulk`, `peek`, `superman`, `headbonk`, `curl`.
→ 런칭엔 `idle` 폴백으로 충분. 여유되면 추가 제작.

`S1`은 플랫/선명 벡터, `S2~S8`은 소프트 도들 — **섞이면 안 됨. 한 스타일로.**
(다수가 소프트 계열)

## 마무리 절차 (파일 주면 기계적으로 끝)

1. 최종 시트 PNG들을 `src/assets/reference/` 에 넣는다.
2. 시트 **픽셀 크기 / 프레임 격자**를 알려주면 내가 `layout.json` 작성
   (또는 `grid` shorthand). → `npm run slice` → `src/assets/cat/<키>/<n>.png` 생성.
3. 내가 `Cat.tsx` 폴백·프레임수 수정 + `catSkin` 스킨 선택/기본값 배선.
4. 빌드 → 맥에서 확인.
